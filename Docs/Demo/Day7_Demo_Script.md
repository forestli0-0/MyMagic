# Day7 Demo Script (10-15 min)

Date: 2026-02-16
Target: 完整展示 Town -> Field -> Boss -> 回城 -> Continue 流程

## 1. 录制前检查（1 分钟）
- 场景与入口：
  - 从 `Assets/Scenes/MainMenu.unity` 进入 Play
  - Build Settings 中确保 `MainMenu/Town/Field/Boss` 已启用
- 数据与运行链路：
  - Day4/Day5/Day6/Day7 Setup 已执行
  - `HUD_Default` 中 CastBar/CombatLog/FloatingText 开关正常
- 录制设置：
  - 1080p 或 1440p，60 FPS
  - 游戏声音开启（用于命中音/施法音反馈）

## 2. 主流程脚本（建议时长 12 分钟）

### 00:00 - 01:00 主菜单与进入游戏
- 打开游戏，展示主菜单。
- 选择 `New Game` 进入 Town。
- 口播点：
  - 展示主循环入口已贯通。
  - 当前角色 HUD 与基础状态正常加载。

### 01:00 - 03:00 Town：任务接取与商店交互
- 与任务 NPC 交互，接取主线/支线。
- 打开任务界面（或观察任务追踪）确认状态变化。
- 与商人交互，做一次买入或卖出。
- 口播点：
  - 任务更新提示出现（toast + tracker 脉冲）。
  - 商店买卖链路完整，货币同步更新。

### 03:00 - 07:30 Field：战斗反馈与掉落反馈
- 进入 Field，触发一波遭遇战。
- 连续施放技能（1~6）并拾取掉落。
- 口播点：
  - 命中反馈：闪白、受击音、相机轻抖。
  - 技能反馈：技能槽施法脉冲。
  - 掉落反馈：拾取 toast（金币/物品）与任务进度提示。

### 07:30 - 10:30 Boss：机制与可读性
- 进入 Boss 场景，完成一场 Boss 战。
- 展示 Boss 技能节奏（telegraph）与玩家应对。
- 口播点：
  - Encounter + BossSkillScheduler 链路可运行。
  - 战斗反馈不会破坏可读性（抖动强度可控）。

### 10:30 - 12:00 回城、交付、保存与继续
- 回到 Town，交付任务并领取奖励。
- 执行一次保存，然后回到主菜单点 `Continue`。
- 验证位置/任务/货币/背包状态恢复正确。
- 口播点：
  - 主线闭环完成。
  - 存档恢复关键状态一致。

## 3. 关键验收点（录制中必须出现）
- `Town -> Field -> Boss` 至少一次完整跳转。
- 任务链：接取 -> 推进 -> 提交。
- 商店链：买或卖至少一次。
- 战斗反馈：闪白/命中音/镜头抖动/技能脉冲。
- 掉落与提示：金币或物品拾取提示至少一次。
- Continue 恢复验证：至少一次。

## 4. 出问题时的补录顺序（最小成本）
1. 先补 `Continue` 恢复镜头（最关键）。
2. 再补 `Field` 战斗反馈与掉落提示。
3. 最后补 Town 的任务交付与商店镜头。

## 5. 交付建议
- 最终视频长度控制在 8-12 分钟（原始录制可 12-15 分钟）。
- 保留 10-20 秒结尾总结画面（展示当前版本完成范围与下一步）。
